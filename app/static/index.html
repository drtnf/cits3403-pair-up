South Pacific Competitive Programming Training Camp

Competitive programming is a combination of mathematics, coding, and problem solving. Given a textual description of a problem, and an input specification, a team must write a program to solve that problem using limited computational resources.

There are many programming competitions held every year including Google Codejam, TopCoder, the International Olympiad in Informatics and the Inter Collegiation Programming Contest (ICPC). In the lead up to the South Pacific Regional of the ICPC we will be holding a short three day training camp, on competitive programming and the tools and processes used in the ICPC.

When?
19-21 January 2021, 9-1 (AWST), 11.30-3.30 (ACDT), 12-4(AEDT) and 2-6 (NZDT).
Where?
All online, with tutorials delivered through Zoom and virtual contest environments.
Who?
This camp is aimed at high school students and university students new to programming contests, or people who want to get practice and learn more about the SPPC online environment. You should be familiar with Python 3, or a similar programming language, and enjoy problem solving.
Why?
You will recieve coaching tips from former world finalists in the ICPC, as well as the regional judging team. You will also help us test and refine our online competitions in preparation for the SPPC regional. Also, all people who attend the camp will recieve an invitation to the second division of the South Pacific Regional Programming Contest, held on January 23.

Limits

  • The time limit is different for each problem.
  • The memory limit for each problem is 1 gigabyte.
  • The source code limit is 256 kilobytes.

Compilation Specifications


C++
Version:gcc 8.3.0
Compilation:g++ -x c++ -Wall -O2 -static -pipe -o prog prog.cc
Execution:./prog < input > output

Java
Version:openjdk 11.0.4
Compilation:javac -encoding UTF-8 -d . prog.java
Execution:java -Dfile.encoding=UTF-8 -XX:+UseSerialGC -Xss128m -Xms${MEMORY_LIMIT}k -Xmx${MEMORY_LIMIT}k < input > output

Python 3
Version:3.7.3
Compilation:python3 -m py_compile prog.py3
(Just used to check for Compile Errors, not in the execution of the code)
Execution:python3 prog.py3 < input > output
C
Version:gcc 8.3.0
Compilation:gcc -x c -Wall -O2 -static -pipe -o prog prog.c -lm
Execution:./prog < input > output
